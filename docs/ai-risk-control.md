# AI 风险与产品对策
| 风险 | 典型表现 | 产品机制 | 评测指标 |
| --- | --- | --- | --- |
| 幻觉 | 模型编造不存在的规范条文或项目结论 | 依据不足拒答；回答只基于检索证据 | 拒答准确率、完整答案通过率 |
| 引用不支持结论 | 回答看似正确，但引用片段无法证明结论 | 引用卡片、Top5 命中校验、引用准确率评测 | 引用准确率 |
| 表格数值错误 | PDF 表格解析乱序，模型输出错误数值 | RagTable 一级对象、表格切片、数值回归题 | 表格数值准确率 |
| 权限泄露 | 用户问到无权项目资料，系统泄露摘要或引用 | 检索前权限过滤；无权资料不进上下文 | 权限隔离通过率、越权泄露率 |
| 资料过期 | 旧版文件被当作当前依据 | 文档生效日期、版本重排信号 | 版本优先命中率 |
| 范围越界 | 用户要求直接判断能否审批通过 | 范围判断和拒答模板 | 超范围拒答准确率 |

## 产品原则

1. 能不回答时不编造。
2. 能引用时必须可追溯。
3. 表格数值由程序渲染，LLM 只负责解释。
4. 权限控制发生在检索前，不在回答后打码。
5. 每个高风险能力都要有可回归的评测题。
