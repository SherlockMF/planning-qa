# One-Pager｜企业知识库问答系统

## 一句话
给城市规划与建筑设计院做一个**会拒答、不越权、能溯源**的企业知识库问答助手——问规范条文和项目资料，像问一位记得住每页原文、且严格遵守权限的资深同事。

## 为谁解决什么问题
**用户**：设计院的普通员工、项目负责人、知识管理员。
**痛点**：
1. 规范/标准分散在大量 PDF/Word，关键指标常埋在表格里，关键词搜不准。
2. 通用 AI 会编造不存在的条款，转述还会漏掉前提条件。
3. 项目资料需按项目隔离，传统网盘做不到「检索那一刻」就过滤，存在越权风险。

## 我们的解法（差异点）
- **抽取式而非生成式**：结论只来自检索命中的原文，依据不足就明确拒答。
- **表格当一级公民**：表格独立存储、命中后由程序精确渲染，LLM 不手写数字。
- **权限前置**：过滤发生在检索之前，无权资料根本不进入 LLM 上下文，也不出现在引用里。

## 为什么是现在
- 设计院数字化诉求上升，但对「AI 编造」零容忍，正好契合「可信 RAG」定位。
- OpenAI 兼容接口成熟（默认接智谱 GLM），无 Key 时可用 mock 全链路演示，落地门槛低。

## 成功长什么样
- 评测题库通过率持续提升且回归不下降。
- 越权泄露率 = 0；引用可溯源率 = 100%。
- 该拒答的拒答、该作答的作答，双向都不误判。

## 范围边界（本期不做）
真实多租户账号、持久化数据库、生产级鉴权、CI 门禁——当前定位为**可信问答内核 + 可演示 demo**，上述为后续里程碑。

## 主要风险
- 表格抽取依赖 Python(pdfplumber) 边车，缺失则降级。
- 无持久化，重启丢数据，暂不可直接上生产。
